New Releases

When there are new releases of Polar, it's highly recommend that you upgrade your own packs. New releases will contain a lot of bug fixes, and probably some new shiny features. The important thing to remember during upgrades is that you want to update everything in the code base but maintain all of the custom configuration you had before.

  1. Copy all of the code (Java) files from the new release into your project (overwrite everything). You'll need to refactor the package name again, as mentioned in Getting Started.

  2. Make a backup of the dev_ prefixed XML files in /res/values.

  3. Copy all of the resource files (everything inside of /res) from the new release into your project, but don't overwrite /res/drawablenodpi or /res/xml

  4. Put back custom configuration from the dev_ prefixed files in /res/values, but be careful to not remove any new values that were added to those files for the upgrade.

  5. Copy over the build.gradle files (from the root of the project and inside the app module). Make sure you put back your applicationId and versionName/versionCode values.

  6. Copy over the AndroidManifest.xml file, put your package back at the top, along with any other custom configuration.

This is less complicated than it sounds, at least when you get used to it. Worst case scenario, just download the project again and start fresh.
