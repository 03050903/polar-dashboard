New Releases

When there are new releases of Polar, it's highly recommend that you upgrade your own packs. New releases will contain a lot of bug fixes, and probably some new shiny features. The important thing to remember during upgrades is that you want to update everything in the code base but maintain all of the custom configuration you had before.

  1. Copy all of the code (Java) files from the new release in your project (overwrite everything).
  2. Copy all of the resource files (everything inside of /res) in your project, but don't overwrite /res/drawablenodpi, /res/xml, or /res/values. It's likely some files in /res/values need updates, but avoid overwriting your own configuration in there.
  3. Copy over the build.gradle files (from the root of the project and inside the app module). Make sure you put back your application ID and versionName/versionCode.
  4. Copy over the AndroidManifest.xml file, put your packageName back, along with any other custom configuration.
